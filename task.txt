# Тестовое задание

## Что нужно сделать

В этом задании вам нужно создать интернет-магазин с динамической корзиной. Ваше приложение будет управлять содержимым корзины (товары и их количество), рассчитывать итоговую стоимость заказа и применять различные скидки к итоговой сумме.

Кроме того, вам необходимо будет подготовить тестовые данные для наполнения приложения: сами товары и скидки.

Реализация прочего функционала интернет-магазина оцениваться не будет, ключевое в задании - именно корзина. Остальное нужно только для наполнения и общего внешнего вида.

## Что нужно сделать, но конкретнее

- Разработать базу данных для каталога интернет-магазина
- Разработать простую страницу каталога, на которой можно будет добавлять товары в корзину
- Реализовать базовый функционал корзины: товары должны добавляться, убираться, а также должна быть возможность редактировать их количество в корзине
- Реализовать подсчет общей суммы заказа (и заложить возможность применять систему скидок)
- Добавить возможность создать заказ из корзины (в рамках тестового не нужно прописывать логику после оформления заказа, теоретически пользователь может хоть сразу оформить следующий заказ)
- Реализовать систему скидок в следующих форматах: скидка на определенный товар (если он есть в корзине - применять скидку), скидка на определенную сумму заказа (если сумма превышает какое-то значение - применять скидку) и комбинированная скидка, по типу товара и сумме заказа (если в корзине есть определенный вид товара и сумма заказа превышает определенный порог - применять скидку)
- Приложение должно уметь хранить данные достаточно долго (дольше, чем обычная сессия) - чтобы обеспечить возможность пользователю собрать корзину и увидеть ее снова после входа через несколько часов (причем авторизация не предполагается, обратите на это внимание)
- Наконец, приложение должно быть протестировано минимум на 70%. Для написания тестов можно пользоваться любым удобным фреймворком - PHPUnit/Pest/Codeception

## Как сдается задание

Готовую реализацию необходимо развернуть на любом удобном вам сервере (бесплатном, разумеется), приложение должно быть доступно из интернета. Кроме того, нужно прислать ссылку на репозиторий с кодом. На репозиторий нужно обратить особое внимание, поскольку качество его ведения (коммиты, реквесты, наличие документации и т.д.) тоже будет оцениваться.

## Как будет оцениваться задание

1. Базовый функционал - приложение доступно, товары добавляются в корзину, заказ создается
2. Расширенный функционал - разработана система скидок, данные о текущей корзине хранятся после выхода пользователя из приложения и могут быть восстановлены
3. Тестовое покрытие - функционал покрыт тестами на 70% или больше
4. Git - код документирован, есть описание - как развернуть приложение, чистая история коммитов (или она хотя бы есть)
5. Время - оценивается скорость выполнения задания относительно предоставленных сроков и качества результатов

## Стек

Приложение нужно написать на Laravel, Symfony или Yii2. Выбор БД, способа деплоя и прочих инструментов целиком на ваше усмотрение.
